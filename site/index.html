Cosmogenesis Data Brain

Static CDN for shared nodes, decks, and hero assets consumed by each spoke.

Live dataset endpoints

Versioned JSON is served with permissive CORS and a one hour cache. Spokes should pin to a versioned path.

  • /data/v1/nodes.min.json - core node registry (<= 50 KB).
  • /data/v1/decks.min.json - optional deck groupings.

Test from a spoke or terminal:

curl -s https://cosmogenesis-learning-engine.netlify.app/data/v1/nodes.min.json

Usage pattern

Each spoke ships the loader script that reads #brain-config from HTML, then fetches the selected version. If the fetch fails it falls back to the inline JSON block for offline review.

Deploy order reminder:

  1. Deploy the brain first and confirm the dataset returns JSON.
  2. Deploy each spoke; they will hydrate from the configured origin.
  3. Offline contexts silently use the inline fallback.

Update workflow

Keep datasets human-auditable: edit the source JSON, then regenerate the minified file if required. Bump the version folder when breaking schema changes are introduced.

All assets stay below the agreed budgets: hero <= 200 KB, JSON <= 50 KB. No embedded media or scripts.
